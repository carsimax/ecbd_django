¿Por Qué Clasificar los Datos por Estructura?

  • Facilita el Procesamiento: Diferentes estructuras requieren diferentes herramientas y técnicas.
  • Determina la Complejidad: Analizar una tabla es muy diferente a analizar texto libre o imágenes.
  • Impacta en el Almacenamiento: Relacional vs. NoSQL vs. Data Lakes.
  • Guía la Selección de Herramientas: SQL, Pandas, NLP, visión por computadora, etc.
  • Entender la estructura es el primer paso para poder extraer valor.
Notas del orador
  • No todos los datos son iguales. La estructura importa.
  • Esta clasificación nos ayuda a organizar nuestro pensamiento y nuestro enfoque técnico.

Tipo 1: Datos Estructurados

Definición: Datos que se adhieren a un modelo predefinido y fijo, organizados en tablas.
Características:

  • Esquema rígido y bien definido.
  • Fácil de consultar y procesar con SQL, hojas de cálculo, Pandas.
  • Representan ~20% de los datos mundiales.
Ejemplos:
  • Bases de datos relacionales (MySQL, PostgreSQL, SQL Server).
  • Archivos CSV, Excel.
  • Registros de ventas, inventarios, CRM.
Notas del orador
  • Estos son los datos más familiares y la base de los sistemas empresariales tradicionales.
  • Pandas está especialmente diseñado para trabajar con este tipo de datos.

Tipo 2: Datos Semi-estructurados

Definición: Datos que no se ajustan a un modelo formal, pero contienen etiquetas o marcadores.
Características:

  • Esquema flexible o implícito.
  • Contienen metadatos o información sobre su estructura.
  • Más fáciles de procesar que los no estructurados, pero requieren parsing.
Ejemplos:
  • JSON, XML.
  • Archivos de log con formato.
  • Emails, datos de NoSQL (MongoDB).
Notas del orador
  • Punto intermedio: tienen estructura interna aprovechable.
  • Muy comunes hoy en día por la web y las APIs.
  • Pandas puede leer JSON y XML, aunque a veces requiere manipulación extra.

Tipo 3: Datos No Estructurados

Definición: Datos sin estructura interna identificable.
Características:

  • Formato libre, difíciles de procesar con herramientas tradicionales.
  • Requieren técnicas avanzadas (NLP, visión por computadora, audio).
  • Constituyen ~80% de los datos generados actualmente.
Ejemplos:
  • Texto libre (emails, redes sociales, documentos).
  • Imágenes, video, audio.
  • Datos crudos de sensores.
Notas del orador
  • La mayor parte de la información del mundo está aquí, pero es la más difícil de analizar.
  • Este curso se centra en estructurados y semi-estructurados, pero es vital conocer este universo.

Comparativa Rápida: Estructura de Datos

Característica Estructurados Semi-estructurados No Estructurados
Organización Alta (Tablas, filas, columnas) Media (Tags, jerarquías) Baja o ninguna

Ejemplos SQL DB, CSV, Excel JSON, XML, Logs Texto, Imágenes, Audio, Video

Notas del orador
  • La línea entre semi-estructurado y no estructurado puede ser difusa.
  • La distinción principal es si hay alguna guía interna para interpretar el contenido.

Fuentes de Datos: ¿De Dónde Vienen?

  • Por Origen: Internas vs. Externas.
  • Por Generador: Humanos vs. Máquinas/Sensores.
  • Por Sector: Negocios, Ciencia, Gobierno, Personal, etc.
  • La clave es la DIVERSIDAD de fuentes.
Notas del orador
  • Piensen en todas las interacciones digitales y físicas que generan datos hoy en día.

Ejemplos Reales de Fuentes de Datos

  • Sistemas Transaccionales (OLTP): Bases de datos de ventas, ERPs, CRMs, bancos. (Estructurado)
  • Actividad Web y Móvil: Logs de servidores, clics, Google Analytics. (Semi-estructurado y estructurado)
  • Redes Sociales: Twitter, Facebook, Instagram. (No estructurado y semi-estructurado)
  • Internet de las Cosas (IoT) y Sensores: Wearables, sensores industriales, GPS, cámaras. (Semi-estructurado o no estructurado)
  • Investigación Científica: Secuenciadores de ADN, telescopios, simulaciones. (Todos los tipos)
  • Datos Públicos y Abiertos: Portales gubernamentales, datos meteorológicos, censos. (Estructurado o semi-estructurado)
Notas del orador
  • Una misma fuente puede generar múltiples tipos de datos.
  • El IoT y la ciencia generan grandes volúmenes y variedad de datos.

Relación Fuentes y Tipos de Datos

No hay una regla 1 a 1: Una fuente puede generar múltiples tipos de datos.
Ejemplo: Una compra online genera:

  • Estructurado: Registro en la BD de ventas.
  • Semi-estructurado: Log del servidor web (JSON).
  • No estructurado: Comentario o reseña del cliente (texto libre).

El desafío: Integrar y analizar datos de diversas fuentes y estructuras.

Notas del orador
  • Es crucial entender qué tipo de datos esperar de cada fuente para planificar su captura y procesamiento.

Implicaciones para la Preparación y Análisis

  • Datos Estructurados: Limpieza, consistencia, joins. Herramientas: SQL, Pandas.
  • Datos Semi-estructurados: Parsing, aplanar jerarquías, limpieza. Herramientas: Pandas, XML/JSON libs, regex.
  • Datos No Estructurados: NLP, visión por computadora, audio. Herramientas: NLTK, spaCy, OpenCV, TensorFlow, PyTorch.

Conocer el tipo y la fuente guía la estrategia de preparación.

Notas del orador
  • El tipo de dato dicta el tipo de trabajo de preparación.
  • Nos centraremos en estructurados y semi-estructurados con Pandas.

Conclusiones Clave

  • Los datos se clasifican en estructurados, semi-estructurados y no estructurados.
  • La mayoría de los datos generados hoy son no estructurados.
  • Los datos provienen de una enorme variedad de fuentes.
  • Comprender el tipo y la fuente es esencial para definir la estrategia de preparación, almacenamiento y análisis.
